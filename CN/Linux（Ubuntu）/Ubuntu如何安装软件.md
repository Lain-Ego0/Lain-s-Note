

# Ubuntu 系统 .deb 包安装指南
## 核心说明
.deb 是 Debian/Ubuntu 系系统的原生安装包格式，以下是 **最常用、高效且避坑的安装方法**，按优先级排序。


### 1. 推荐方法：apt 直接安装（自动处理依赖）
#### 适用版本
Ubuntu 16.04 及以上（含 20.04、22.04、24.04 等主流版本）
#### 命令
```bash
sudo apt install /path/to/your/package.deb
```
#### 关键说明
- 替换 `/path/to/your/package.deb` 为文件实际路径（例：`~/Downloads/robot_tool.deb`）；
- 无需额外手动处理依赖，apt 会自动检测并安装缺失的依赖包，成功率最高；
- 执行前建议更新 apt 源：`sudo apt update`，避免因源缓存导致的安装失败。


### 2. 基础方法：dpkg 安装（需手动修复依赖）
#### 核心作用
Debian/Ubuntu 原生包管理工具，直接操作 .deb 包
#### 命令
```bash
sudo dpkg -i /path/to/your/package.deb
```
#### 参数解释
- `-i`：全称 `install`，表示“安装指定包”；
- 路径替换规则同方法 1，可直接拖入文件到终端自动填充路径。
#### 必看注意：处理依赖报错
若执行后提示 **“缺少依赖”“dependency errors”**，立即执行以下命令修复：
```bash
sudo apt install -f
```
`-f`：全称 `fix-broken`，表示“修复损坏的依赖关系”，自动补装缺失的依赖；修复完成后，若包未自动完成安装，重新执行 dpkg 安装命令即可。


### 3. 图形化工具（适合新手，注意闪退问题）
#### 操作步骤
1. 找到 .deb 文件（如 Downloads 文件夹）；
2. 双击文件，系统会自动调用 **Ubuntu Software** 打开；
3. 点击安装，输入系统密码即可。
#### 避坑提醒
已知问题：Ubuntu 18.04、20.04、22.04 版本均出现过双击闪退现象，是系统自带工具兼容性问题。若闪退，直接切换到上述命令行方法。效率更高且无兼容性问题。

### 常见问题排查
- 若提示权限不足：命令前必须加 `sudo`获取管理员权限；
- 若提示文件不存在：检查路径是否正确。终端中拖入文件自动填充路径，避免手动输入错误；
- 若依赖修复失败：执行 `sudo apt update && sudo apt full-upgrade` 后，重新尝试安装。
